# 亲亲，这边不建议呢

一个内容驱动的客服模拟游戏原型。玩家在客服后台里处理不同客户会话，在满意度、怒气、公司成本、合规风险和剩余时间之间做取舍。

## 开发目标

- 先搭建可维护框架，而不是单文件 demo。
- 客户、话术卡、知识库、关卡等内容独立维护。
- 游戏状态流转集中在 `src/game`，UI 组件只负责展示与触发动作。

## 目录

```text
src/
  content/      游戏内容数据
  game/         类型、状态机、评分和结算
  components/   客服后台 UI 组件
  styles/       全局样式
```

## 常用命令

```bash
npm install
npm run dev
npm run build
```
